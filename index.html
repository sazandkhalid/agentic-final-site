CS 6725 · Final Project

Self-Constructing Agents with Verified Skill Persistence

Procedural memory in code-acting language agents through a four-gate verification pipeline

Overview

Most language-model agents are stateless — every task starts at zero, and engineering knowledge produced in one session evaporates when it ends. In domains where the same problems recur, a stateless agent is structurally incapable of improving at the work.

This project presents an architecture in which an agent writes, verifies, and accumulates a permanent library of executable Python skills as a by-product of normal task execution, then reuses them across future sessions through composition, evolution, and decay.
